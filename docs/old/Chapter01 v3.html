ProofTree is a syntax and system that allows you to write proof of mathematical statements in a rigourous way, and to confirm that those proofs are valid. What is a proof? A proof is a series of steps that together derive the truth a specific statments. Consider the following proof that the lawn is wet.

It is raining. If its rains, the lawn will be wet. Therefore, the lawn is wet.

This proof looks pretty simple. However, nuances of the english langauge are hiding the fact that there is more to the proof than meets the eye. Let's break it down so we can see exactly what's going on.

The following satements are coherent (though not necesarily true): It is raining. The lawn is wet. We will accept the following as true: (1) It is raining. (2) If its rains, the lawn will be wet. We will now prove that the lawn is wet: By (1), it is raining. Thus, by (2), the lawn is wet.

Now that we've deconstructed the proof and understand exactly all of the pieces involved, we can write this proof using ProofTree. To start, open a new file (Notebook is fine, but you can download and use SublimeText if you want to be fancy) and name it Main.proof. Let's start translating!

English The following satements are coherent (though not necesarily true): It is raining. The lawn is wet.

ProofTree opaque predicate R opaque predicate W

These two lines create two new predicates, or statements about the world. The opaque identifier simply means that we don't yet know anything about the predicates (such as whether or not they are true). We will use the predicate R to mean "it is raining," and the predicate W to mean "the lawn is wet."

English We will accept the following as true: (1) It is raining. (2) If its rains, the lawn will be wet.

ProofTree axiom rain : R axiom rain_lawn : R -> W

An axiom is a statement that we will assume is true. When declaring a new axiom, we need rain_lawn things: the name of the axiom, so we can refer to it later, and the statement that we are asserting. The former goes before the colon, while the latter goes after it. Note that statements of the form "if A then B" get translated to "A -> B."

English We will now prove that the lawn is wet:

ProofTree theorem : W {

Pretty straightfoward: new proofs begin with the theorem identifier, followed by a colon, followed by the statement to be proved, and then a opening curly brace.

English By (1), it is raining.

ProofTree R | rain

Every step of a ProofTree proof begins with the statement being assserted at that step, and concludes with the axiom that justifies it, separated by the pipe symbol. Similarly,

English Thus, by (2), the lawn is wet.

ProofTree W | rain_lawn

Finally, we conclude the proof with a closing curly brace. Putting it all together, our first ProofTree proof looks like this:

opaque predicate R opaque predicate W axiom rain : R axiom rain_lawn : R -> W theorem : W { R | rain W | rain_lawn }
